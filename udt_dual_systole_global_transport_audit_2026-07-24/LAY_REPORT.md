# Lay report

The previous work found a possible built-in “direction” inside the angular
part of the metric. This audit stopped sampling a few examples and mapped
that object everywhere it can exist.

Imagine that each torus-like angular slice has a woven grid. The metric tells
us which primitive loop in the **dual** grid is shortest. Usually there is
one shortest direction. Along exact dividing walls, two directions are tied.
Where those walls meet, three are tied. The mathematics proves there can
never be four tied shortest directions.

That pattern is not an arbitrary numerical discovery. The entire pattern is
one basic wall repeated by exact changes of grid coordinates. It survives a
change of torus coordinates, so it is genuine geometry rather than a chart
artifact.

This also clarifies the reciprocal branch. On one side of `phi=0`, one
angular direction is shortest. On the other side, its reciprocal partner is
shortest. Exactly at zero they tie. The metric itself faithfully supplies
the pair, but does not point to one and say, “continue this one as the
physical phase.”

So we gained a real piece of native structure:

- the metric supplies a precise, globally transportable **set** of preferred
  geometric directions on toric branches;
- on a branch that never touches a tie, that set reduces to one unoriented
  line and gives a natural one-dimensional connection;
- caps, twists, mirrors, and rank loss tell us exactly when that object can
  or cannot extend.

What we did **not** gain is equally important. Shortest does not automatically
mean physical. We still do not have a native rule choosing a sign, a phase
section, a carrier, or an action. The result narrows the missing step: if UDT
does choose a physical phase direction, the choice must explain what happens
at the exact tie walls and through the finite-cell completion.
