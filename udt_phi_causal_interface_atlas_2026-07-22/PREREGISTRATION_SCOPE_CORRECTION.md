# Preregistration Scope Correction — Identity Versus Presentation and Silent Phi

Date: 2026-07-22

Status: `PREREGISTERED_BEFORE_PRODUCTION_RESULT`

The frozen generator/schema inspection after the main preregistration reveals three counting and
domain facts that must be fixed before production:

1. The structural-mask registry includes identities for which the `PHI_FIELD` bit is absent. For
   those identities `dphi` is identically zero on the complete path. They are classified once as
   `IDENTICALLY_ZERO_DPHI_INTERVAL`; they are not represented as infinitely many isolated
   `ZERO_GRADIENT_EVENT` roots.
2. `s=g^{-1}(dphi,dphi)` is determined by the analytic identity and coordinate point, not by which
   instrument-family subset is used to present that metric. Root certification is therefore
   performed once for each of 3,072 identities and then joined mechanically to all 31 family
   presentations. Identity counts and presentation counts remain separate everywhere.
3. The source registry labels every path `LOCAL_ANALYTIC_CHART_ONLY`. Its straight coordinate chord
   is the full registered path interval, not a complete finite-cell geometry. Crossing the results
   with the twelve global taxonomy rows may state compatibility requirements only; it cannot create
   a complete finite-cell witness.

The production verifier must fail if a silent-phi interval is converted to isolated roots, if one
identity-level event is counted as 31 independent events, or if a local path is promoted to a
global completed geometry.

No candidate identity, family presentation, root, or completion row is removed by this correction.
