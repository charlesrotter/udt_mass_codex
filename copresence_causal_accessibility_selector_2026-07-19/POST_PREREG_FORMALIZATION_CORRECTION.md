# Post-Preregistration Formalization Correction

Date: 2026-07-19  
Historical preregistration commit: `22b7f08`

The committed preregistration wrote events as if they were elements of a field solution `S`. That is
type-imprecise. The preregistration remains unchanged as historical evidence. All result and
verification records use the corrected definition:

\[
S=(M_S,g,\Phi,\ldots),
\qquad
p\,C_S\,q \Longleftrightarrow p,q\in M_S=\operatorname{dom}(S).
\]

Thus co-presence is shared event-domain membership in one whole candidate solution. It is ordinary
block-universe co-membership unless additional UDT structure is separately derived. It contains no
causal, simultaneous, metric, or action selector by itself.

The preregistered phrase “causal partition” is also retained only as lay shorthand. The technical
object is the event-relative causal reachability relation `J_g^+`. It is not an equivalence partition,
and it becomes a causal order only under additional global causality conditions excluding causal
loops.

These corrections narrow terminology; they do not change the registered outcome set, tested model
classes, expected outcome, or falsification contract.
