# Stop and next scientific decision

Numerical time-live evolution is not yet the next selector test. Every evolution code must be given
a boundary prescription, and the present audit proves that several inequivalent prescriptions
survive. A successful evolution would therefore certify only the chosen conditional branch.

The smallest remaining conceptual question is whether the finite-cell boundary is a **free global
embedding** rather than a prescribed wall:

```text
Does the existing statement that X_max and the realized finite cell are global outputs
authorize variation of the seal embedding, and if so, does the resulting transversality
condition jointly fix its causal type, allowed field variations, and boundary functional?
```

That is narrower than inventing a new action and more direct than a PDE search. It must be treated
as an owner-status/provenance question first: the current bootstrap is on-shell, and promoting the
cell embedding to an off-shell varied object is not presently derived.

If that extension is not authorized or remains nonunique, the honest numerical route is a
comparative time-live atlas across every surviving boundary class. It could map existence,
constraint preservation, and global regularity, but it could not call the winning branch native
without a separately stated selection rule. Matter-bearing bootstrap filtering would additionally
require the still-open native source/carrier and total-density functional.

No free-boundary audit, P06 solve, time integration, GPU work, or new premise is launched here.
