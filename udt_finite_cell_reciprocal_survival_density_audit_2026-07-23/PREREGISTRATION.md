## Density-interface gate

A density-dependent branch scan is authorized only if current UDT supplies
at least one of:

```text

```

with exact provenance, or a simultaneous closure containing a native
off-shell mass functional, same-solution proper volume, varied global
closure, and a response map from `rho_tot` to the branch equations.

Definitions such as `rho_tot=M_tot/V_tot`, dimensional rearrangements, an
observed density, and conditional carrier readouts do not pass this gate.

If the gate passes, the density schedule and units must already follow from
the sourced interface; no desired answer may set the range. Branches will be
characterized as appearing, disappearing, merging, changing causal class,
becoming singular, or persisting.

If the gate fails, no numerical density sweep will be performed. Instead the
audit must test the stronger formal bracket `rho_tot in [0,infinity)` and
record that, in the audited geometry-only system, the branch map is constant
because `rho_tot` is not an argument of any branch equation. This is a
stopping theorem about the current interface, not a claim that physical
density is irrelevant.

  complete metric solution;

  throughout the supplied domain;

- a density normalization, source, action, or mass functional is invented;
- the 12 registered completion rows are filtered toward a desired topology;
- frozen source identities, current paths, or repository gates change.

Certification requires deterministic production output, a structurally
independent verifier, exercised catches, source replay, a fresh adversarial
