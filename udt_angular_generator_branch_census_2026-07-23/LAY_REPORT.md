# Lay report

We asked whether the angular part of every possible complete metric branch
naturally develops the same “one side shrinks while the other grows” pattern
as the reciprocal clock/ruler backbone.

The answer is: **the metric gives us an exact test, but none of the currently
registered global branches automatically passes it.**

The angular sector can do two separate things as `phi` changes:

1. its total two-dimensional area can grow or shrink; and
2. it can change shape—stretching one angular direction while squeezing the
   other.

To match the reciprocal backbone exactly:

- the angular area must remain constant relative to the clock/ruler block;
  and
- the shape change must occur at exactly the reciprocal unit rate.

One registered family, the reciprocal-toric branch, always has the correct
opposite-direction shape change. But it still allows its total angular size
to vary freely. If that size is held constant relative to the reciprocal
block, the match is exact. If it varies, both angular rates are shifted
together and the match is lost.

That is useful progress. The remaining question is no longer a vague
“perhaps the angular sector couples somehow.” It is:

> Does the complete metric derive constant relative angular area and unit
> angular shape speed?

The global branch catalogue does not yet answer yes:

- cap branches lose the two-dimensional angular metric at their caps;
- periodic branches force `phi` to have a stopping point unless extra cover
  structure is supplied;
- most mirror and monodromy choices do not preserve the reciprocal
  directions; and
- most branch definitions specify topology, not the actual metric profile.

The observed `c` remains explicitly present in the complete metric. It fixes
the clock/ruler conversion but does not fix angular size or its rate of
change.

So we found a clean geometric target already latent in the metric, not a new
mechanism. We have not yet found the native rule that forces it.
